# 04-RESSOURCES

Ressources transversales de SCC.

## Rôle
Ce dossier sert à centraliser les ressources utiles qui ne sont pas rattachées à un seul projet actif.

Il peut contenir :
- outils de travail
- références réutilisables
- docs de support
- cadres généraux
- ressources communes à plusieurs projets ou agents

## Principe
`04-RESSOURCES` contient des éléments utiles, mais non centraux à un projet unique.

Ce n’est pas :
- un dossier projet
- une archive inactive
- une inbox temporaire
- un runtime technique
- un dépôt de fichiers jetés sans logique

## Ce qui a sa place ici
- ressources réutilisables par plusieurs projets
- documents de référence généraux
- outils ou cadres de travail communs
- supports non liés à un seul dossier projet
- matériel utile à plusieurs agents ou plusieurs workflows

## Ce qui ne doit pas vivre ici
- documentation active d’un projet précis
- fichiers temporaires en attente de tri
- archives historiques
- backups
- secrets
- configuration runtime Hermes active
- bruit technique sans valeur documentaire

## Lien avec le reste de SCC
- `01-INBOX` = réception temporaire
- `02-PROJECTS` = projets actifs
- `03-ARCHIVES` = conservation inactive
- `04-RESSOURCES` = ressources transversales
- `05-AGENTS` = règles et cadrage agentique

## Règle de classement
Avant de mettre un élément ici, il faut vérifier :

1. ce n’est pas un fichier temporaire
2. ce n’est pas une archive
3. ce n’est pas la vérité active d’un projet précis
4. ce n’est pas un élément runtime ou secret
5. il garde une utilité transversale

## Règle centrale
Une ressource utile doit être facile à retrouver et clairement classée.

Si un élément ne sert qu’à un seul projet, il doit plutôt vivre dans ce projet.

## Règle finale
`04-RESSOURCES` doit aider le travail.
Pas créer un tiroir fourre-tout de plus.
