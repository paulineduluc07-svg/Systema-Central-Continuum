# Product Positioning -- 2026-03-22

## Clarification centrale
- Kim n est pas une application de compagnie.
- Kim est un AI agent ultra-competent oriente execution.
- Kim doit aider l utilisateur sur ses fichiers, dossiers, sites web, outils et workflows.
- La proposition de valeur centrale est l amelioration de performance de vie, pas la compagnie.

## Separation des references
- Posture produit / execution / impression generale: reference plus proche d un produit type OpenClaw.
- Frontend visuel / presence / finition: Replika et certaines references jeu video peuvent servir de modele.
- Cette reference visuelle ne change pas la categorie produit.

## Consequences frontend
- L avatar et la scene doivent suggerer competence, clarte, controle et qualite premium.
- Le 3D doit soutenir la confiance et la lisibilite, pas brouiller la proposition de valeur.
- Eviter les codes trop relationnels si cela dilue le positionnement execution-first.
- Les options cosmetiques type wardrobe ou room decoration ne doivent jamais passer avant la clarte produit coeur.

## Consequences backend et produit
- Le backend, les tools et les surfaces utilitaires doivent etre penses comme un systeme d action et d assistance haute competence.
- Les flux auth, permissions, tools et fichiers doivent inspirer confiance, serieux et efficacite.
- Le chat reste important, mais comme interface d action, pas comme preuve de valeur relationnelle.

## Usage des references ajoutees
- Les captures ajoutees dans `Notes/` servent de support de comparaison Kim / Replika pour l audit visuel.
- Elles doivent etre lues comme references de finition frontend, pas comme spec produit complete.

## Rappel
- Le nom du repo peut rester historique.
- La source de verite produit devient ce cadrage de positionnement.
