# Règles Multi-Agents & Amélioration Continue (valable pour TOUT le SCC)

1. Tous les projets sont conçus pour que plusieurs agents puissent accéder et travailler sur les mêmes dossiers/fichiers en même temps ou successivement.

2. Utilise Git correctement : commits clairs, messages précis, branches uniquement si nécessaire (sinon tout sur main).

3. Aucun agent ne doit supprimer ou écraser le travail d'un autre sans validation.

4. Priorité absolue = AMÉLIORATION CONTINUE :
   - Si tu détectes un bug, une erreur, une mauvaise pratique ou une possibilité d'amélioration → tu vérifies, tu analyses, tu proposes une solution.
   - TU NE DOIS JAMAIS appliquer la correction ou l'amélioration sans m'en parler avant (tu me dis exactement ce que tu veux changer et pourquoi).

5. Exemple concret : si on travaille sur le déploiement d'un site web et qu'un agent bugue ou propose quelque chose que je n'aime pas, un autre agent peut immédiatement ouvrir les mêmes dossiers et continuer le travail sans rien casser.
