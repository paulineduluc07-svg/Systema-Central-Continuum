# Règles IPARA

> Système d'organisation : Inbox / Projects / Areas / Resources / Archives

---

## Structure

| Dossier | Rôle |
|---|---|
| `Inbox/` | Capture brute. Vider chaque semaine. Tout nouveau fichier ici par défaut. |
| `Projects/` | Projets avec une ligne d'arrivée claire. In Progress ou On-Hold. |
| `Areas/` | Responsabilités continues sans fin définie. Un `_CONTEXT.md` par area. |
| `Resources/` | Templates, SOPs, prompts IA, références. |
| `Archives/` | Projets terminés. Jamais supprimés, juste archivés. |

---

## Règles absolues

1. **Une seule source de vérité** — ce repo GitHub est la référence vivante. Pas de copies ailleurs.
2. **Zéro doublon actif** — un fichier = un seul endroit = une seule raison d'être.
3. **Les agents ajoutent seulement** — jamais modifier le contenu existant sans validation.
4. **Inbox se vide** — chaque semaine, chaque fichier de l'Inbox est trié ou supprimé.
5. **Archives ≠ poubelle** — un projet archivé garde tout son contenu intact.

---

## Structure d'un projet actif

```
Projects/Nom-Du-Projet/
├── Code/               ← liens/refs vers le repo code (jamais le code ici)
├── Notes/              ← notes de travail, réflexions
├── Notes-Perso/        ← optionnel, privé
├── Prompts/            ← prompts IA spécifiques au projet
├── Assets/             ← images, exports, fichiers statiques
├── Livrables/          ← outputs finaux
├── Todo.md
├── Roadmap.md
└── AGENT-INSTRUCTIONS.md
```
