# 01-INBOX

Zone de réception temporaire de SCC.

## Rôle
Ce dossier sert uniquement à recevoir des éléments entrants avant tri :
- captures
- documents
- exports
- notes brutes
- fichiers de passage
- éléments à classer dans le bon dossier

## Principe
`01-INBOX` n’est pas un dossier de stockage permanent.

C’est une zone tampon courte durée.

Un fichier ici doit rapidement être :
- déplacé vers le bon projet dans `02-PROJECTS/`
- déplacé vers `04-RESSOURCES/` s’il s’agit d’une ressource réutilisable
- déplacé vers `03-ARCHIVES/` s’il est inactif mais à conserver
- supprimé s’il est inutile, doublon, test ou bruit

## Ce qui a sa place ici
- fichiers entrants non encore triés
- éléments reçus pendant une session de travail
- exports temporaires en attente de classement
- documents à examiner avant décision

## Ce qui ne doit pas rester ici
- documentation active durable
- fichiers projet déjà classés
- archives anciennes
- backups
- fichiers techniques parasites
- dossiers temporaires oubliés
- doublons conservés “au cas où”

## Règle de tri
Tout élément dans `01-INBOX` doit être traité avec une logique simple :
1. identifier ce que c’est
2. décider s’il est actif, ressource, archive ou déchet
3. le déplacer ou le supprimer
4. ne rien laisser traîner sans raison claire

## Règle de discipline
`01-INBOX` doit rester petit, lisible et transitoire.

S’il grossit, c’est un symptôme de non-tri.

## Lien avec le reste de SCC
- `01-INBOX` = réception temporaire
- `02-PROJECTS` = travail actif structuré
- `03-ARCHIVES` = conservation inactive
- `04-RESSOURCES` = ressources transversales
- `05-AGENTS` = règles agentiques et cadrage

## Règle finale
Un fichier ne doit pas vieillir ici.
S’il reste dans `01-INBOX`, c’est qu’une décision manque.
