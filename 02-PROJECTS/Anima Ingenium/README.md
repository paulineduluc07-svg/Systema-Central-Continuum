## Règles du Projet
1. **Synchronisation Permanente** : Tout ce qui est configuré sur le VPS (modèles, clés, réglages) doit être reflété ici. La documentation SCC est le miroir de la vérité opérationnelle.
2. **Passes Fermées** : Appliquer strictement le cadre `RESSOURCES/CADRE_TRAVAIL_PASSES_FERMEES.md` pour toute intervention runtime.
3. **Runtime Officiel** : La source de vérité opérationnelle est le VPS décrit dans `RESSOURCES/RUNTIME_SPECIFICATIONS.md`.
4. **Autorisation** : Aucune action sensible sans validation explicite.
5. **Traçabilité** : Toute modification significative doit laisser une trace dans `WORKLOG.md`.
6. **Cloisonnement** : Ne pas mélanger documentation projet et runtime réel.
