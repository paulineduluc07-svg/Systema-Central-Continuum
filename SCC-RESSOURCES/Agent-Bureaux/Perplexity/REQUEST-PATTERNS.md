# REQUEST-PATTERNS -- Perplexity

## Quand utiliser Perplexity en priorite
- Audit de structure, de contenu ou de coherence (SCC, fichiers, projets)
- Recherche d'information actuelle ou technique avec verification
- Discussion strategique et aide a la decision
- Exploration d'une idee en mode libre avant de la formaliser
- Triage de liste de taches ou de priorites
- Redaction ou mise a jour de fichiers directement dans le SCC
- Support emotionnel leger et recadrage quand Paw est en surcharge
- Brainstorming initial avant de passer a Codex pour l'implementation

## Patterns de prompt efficaces avec Perplexity

### Audit
```
Fais un audit de [fichier/dossier/projet] et reviens avec ce que tu observes -- problemes, forces, doublons, manques.
```

### Recherche
```
Cherche [sujet] et dis-moi ce qui est pertinent pour [contexte specifique].
```

### Decision
```
J'hesite entre [option A] et [option B] pour [objectif]. Aide-moi a choisir avec les pour/contre reels.
```

### Mise a jour SCC
```
Mets a jour [fichier] dans le SCC avec [information]. Respecte le format existant.
```

### Discussion ouverte
```
J'ai besoin de parler de [sujet]. Pas de structure forcee -- juste discutons.
```

## Ce qui fonctionne mal
- Demander a Perplexity d'executer du code complexe en local
- Demander un acces direct a des espaces non partages (Notion, OneDrive interne)
- Demander une memoire de sessions precedentes sans fournir le contexte dans les fichiers SCC
